A. Historial
Cargar el entorno y inicializar estructuras:
En este paso, cargas las variables de entorno necesarias para tu shell y
también inicializas las estructuras de datos necesarias para gestionar el historial de comandos.

Mostrar el prompt:
Muestra el prompt (indicador de que el shell está listo para recibir comandos) para
que el usuario sepa que puede ingresar un comando.

Lectura de argumentos (Readline):
Utiliza la función readline para leer la entrada del usuario, incluyendo cualquier
comando que el usuario pueda ingresar.

Análisis de entrada:
Analiza la entrada del usuario, comprobando si hay metacaracteres, comillas u otros
elementos que requieran un manejo especial. Luego, divide la entrada en tokens
(palabras o elementos individuales), elimina los espacios innecesarios y expande las
variables si es necesario.

Comprobación de redirecciones:
Verifica si hay redirecciones presentes en los comandos ingresados por el usuario
y verifica su corrección.

Dividir comandos por pipes:
Si hay tuberías presentes en los comandos, divídelos en comandos individuales para su
posterior ejecución.

Comprobación de tokens en Built-in:
Verifica si los comandos ingresados son built-ins (comandos internos del shell),
 como cd, pwd, echo, etc.

Comprobación de tokens en PATH:
Si no son built-ins, verifica si los comandos están disponibles en las rutas
especificadas en la variable PATH del entorno.

Comprobación si el token es un archivo:
Si el token no se encuentra en las rutas especificadas en PATH, se considera un 
archivo. Se procede a abrir y ejecutar el archivo correspondiente.

Abrir archivos:
Si hay redirecciones o tuberías presentes, abre los archivos correspondientes
según sea necesario.

Ejecución (pipex):
Ejecuta los comandos correspondientes, ya sea built-ins, comandos en las rutas
de PATH o archivos.

Cerrar archivos:
Una vez finalizada la ejecución, cierra todos los archivos que se hayan abierto
durante el proceso.

Volver al paso 2:
Después de completar todas las operaciones necesarias, vuelve al paso 2 para
esperar la próxima entrada del usuario.

B. Interactivos
Los pasos para la interacción con el usuario (mostrar el prompt, leer la entrada,
analizar la entrada, etc.) son los mismos que en la sección A, pero en este caso,
se están describiendo como parte del proceso continuo de interacción con el usuario
en un shell interactivo. La diferencia principal entre las secciones A y B es que la
sección A se enfoca específicamente en el manejo del historial de comandos,
mientras que la sección B describe el flujo general de interacción con el usuario 
en un shell interactivo.
